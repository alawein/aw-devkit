# Voice Guide

How to write copy for Alawein interfaces. The voice is a field manual — precise, confident, slightly mysterious.

## Tone

- **Direct**: Say what it is. No hedging, no "might" or "could potentially."
- **Technical**: Use precise terminology. "Deploy" not "put out there."
- **Confident**: State facts. "This works." Not "we think this should work."
- **Restrained**: Let the design speak. Copy supports, never competes.

## Principles

1. **Brevity over explanation** — If the UI is clear, the label is enough
2. **Operational over marketing** — "Initialize" not "Get started today!"
3. **Evidence over promises** — Show the number, not the adjective
4. **Structure over decoration** — Let hierarchy do the emotional work

## Vocabulary

Prefer:
- Deploy, initialize, transmit, configure, execute
- Operative, system, module, sector, protocol
- Active, nominal, standby, offline, critical

Avoid:
- Launch, boost, supercharge, revolutionize
- User, customer, visitor
- Amazing, incredible, powerful, seamless

## Evidence Marks

Footer-style metadata that proves the system is real:
- Issue numbers: `ISS-001`, `REF-042`
- Timestamps: `2026.03.07`
- Version tags: `v0.1.0`, `BUILD-001`
- Facility codes: `FACILITY-07`, `CABINET-01`

## Examples

Bad: "Welcome to our amazing design system! Get started in minutes!"
Good: "Field Manual Edition 001. Deploy with confidence."

Bad: "Click here to learn more about our powerful features"
Good: "Access briefing"

Bad: "We've built something incredible for you"
Good: "System operational. All subsystems nominal."
